# Assessment Overview

This course evaluates beginner Python learning through guided practice, notebook work, debugging, reflection, and a small mini project.

The goal is to show growing understanding, not to write advanced code.

## Assessment Types

### Guided Labs

Guided labs ask students to complete structured tasks in notebooks. They usually include examples, TODO sections, output checks, and short reflection prompts.

Guided labs help assess:

- following step-by-step instructions
- editing starter code
- running and testing code
- applying one or two Python concepts at a time

### Notebook Exercises

Notebook exercises are smaller practice activities inside the course notebooks. They may ask students to change values, predict output, fix code, or explain what happened.

Notebook exercises help assess:

- active participation
- basic Python syntax
- reading output
- making small revisions

### Mini Project

The guided mini project asks students to build a small Python program using several course skills together.

The project should include:

- input or starter values
- variables with clear names
- at least one conditional
- at least one loop
- at least one list
- arithmetic or simple processing
- clear final output
- testing notes
- a reflection or fix log when required

The mini project should stay small and explainable. A clear beginner project is better than a complex project the student cannot describe.

### Reflections

Reflections ask students to explain what their code does, what they learned, what was difficult, and how the work could apply to school, work, or daily life.

Reflections help assess:

- understanding in the student's own words
- connection between code and purpose
- awareness of debugging and revision
- responsible learning habits

## Grading Philosophy

Assessment in this course should reflect the course philosophy:

- clarity over complexity
- understanding over memorization
- practice over perfection
- debugging as part of learning
- simple working code over advanced unexplained code

One mistake should not automatically ruin an entire submission. For example, a student may have a small syntax error but still show correct reasoning, completed steps, and useful debugging notes.

## Student Expectations

Students are expected to:

1. Complete required notebook sections.
2. Run code and check the output.
3. Fix errors when possible.
4. Explain their work in their own words.
5. Submit work in the requested format.
6. Avoid copying full solutions from another person, website, or AI tool.
7. Ask for help when stuck and describe what they already tried.

Students may be asked to explain their code. This helps confirm that the submitted work reflects their own learning.

## AI Policy Reference

AI tools may support learning, but students remain responsible for what they submit. Students must test AI-assisted code and be able to explain it in their own words.
