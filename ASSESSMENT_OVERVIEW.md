# Assessment Overview

This course evaluates beginner Python learning through guided practice, notebook work, debugging, reflection, and a small mini project.

The goal is to demonstrate growing understanding, not to produce advanced or complex code.

## Assessment Types

### Guided Labs

Guided labs ask students to complete structured tasks within notebooks. These typically include examples, TODO sections, output checks, and short reflection prompts.

Guided labs help assess:

- ability to follow step-by-step instructions  
- editing and completing starter code  
- running and testing code  
- applying one or two Python concepts at a time  


### Notebook Exercises

Notebook exercises are smaller practice activities embedded throughout the course notebooks. They may ask students to change values, predict output, fix code, or explain what happened.

Notebook exercises help assess:

- active participation  
- basic Python syntax and structure  
- reading and interpreting output  
- making small corrections and improvements  


### Mini Project

The guided mini project asks students to build a small Python program that combines multiple course concepts.

The project should include:

- input or starter values  
- clearly named variables  
- at least one conditional  
- at least one loop  
- at least one list  
- arithmetic or simple processing  
- clear final output  
- basic testing notes  
- a reflection or fix log (when required)  

The mini project should remain small, clear, and explainable. A simple, well-understood program is better than a complex one the student cannot describe.


### Reflections

Reflections ask students to explain what their code does, what they learned, what was challenging, and how the work connects to real-world tasks.

Reflections help assess:

- understanding in the student's own words  
- connection between code and purpose  
- awareness of debugging and revision  
- responsible learning habits  


## Grading Philosophy

Assessment in this course follows a beginner-centered approach:

- clarity over complexity  
- understanding over memorization  
- practice over perfection  
- debugging as part of learning  
- simple, working code over advanced but unexplained code  

A single mistake should not automatically invalidate an entire submission. For example, a student may have a small syntax error but still demonstrate correct reasoning, completed steps, and useful debugging notes.


## Student Expectations

Students are expected to:

1. Complete all required notebook sections  
2. Run their code and check outputs carefully  
3. Fix errors when possible  
4. Explain their work in their own words  
5. Submit work in the requested format  
6. Avoid copying full solutions from other students, websites, or AI tools  
7. Ask for help when stuck and describe what they have already tried  

Students may be asked to explain their code. This helps confirm that submitted work reflects their own learning.


## AI Policy Reference

AI tools may support learning, but students remain responsible for what they submit. Any AI-assisted code must be tested, understood, and explained.
