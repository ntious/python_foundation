# Grading Overview

This course evaluates your progress with practical Python work. The focus is on learning to solve small problems, test your code, and explain your thinking.

This page gives a general course-level overview. Your instructor may provide more specific grading details for each assignment.

## What Instructors Look For

Instructors usually look for evidence of:

- completion of required tasks
- code that follows the assignment directions
- correct or reasonable output
- clear use of Python basics taught in the course
- effort to test and revise your work
- explanations that show your own understanding
- care with file names, instructions, and submission requirements

You are not expected to write advanced or professional software. You are expected to practice carefully and show what you understand.

## Correctness

Correctness means your code does what the task asks it to do.

For beginner assignments, this may include:

- using the correct variables or values
- calculating a total or average correctly
- writing an `if` statement that checks the right condition
- using a loop to process each item in a list
- reading a simple data file correctly

Small mistakes can happen. One error should not automatically erase the value of the rest of your work.

## Completion

Completion means you attempted all required parts of the assignment.

This includes code cells, written answers, reflections, fix logs, or explanation prompts when they are required. Blank required sections make it harder to see what you learned.

## Understanding

Understanding matters more than complexity. A short solution that you can explain is better than a complicated solution that you do not understand.

You may be asked to explain:

- what your code does
- why you chose a condition or loop
- what an error meant and how you fixed it
- how you tested your answer
- what you would try next if the code did not work

## Debugging and Testing

Debugging means finding and fixing problems in code. Testing means running code and checking whether the result makes sense.

In this course, debugging and testing are part of the learning process. Assignments may give credit for showing that you:

- read an error message
- tried a reasonable fix
- tested with a small example
- revised your code after seeing unexpected output
- explained what changed

## Following Directions

Following directions is part of professional and academic work. Read assignment instructions carefully before submitting.

Check for:

- required file names or formats
- required notebook sections
- specific data files to use
- reflection or explanation prompts
- limits on AI assistance or outside help

## A Helpful Mindset

Grading is not only about whether code is perfect. It is also about whether your work shows practice, care, and growing understanding.

Aim for code that is:

- simple
- readable
- tested
- connected to the task
- explainable in your own words
