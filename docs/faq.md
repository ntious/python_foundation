# FAQ

## Do I need coding experience before taking this course?

No. This course is designed for beginners. You will start with basic Python ideas and build skills through guided practice.

## What should I do if my code gives an error?

Read the last line of the error message first. Then check the cell where the error happened. Look for spelling, punctuation, indentation, missing quotation marks, or a skipped earlier cell.

Errors are normal. They are part of learning to program.

## What if my output looks different from the example?

First, check whether the difference is expected. Your name, values, or data may be different from the example.

If the output should match, rerun the previous cells in order and compare your code to the instructions. Make one small change at a time.

## Can I use AI for help?

Yes, when it supports your learning. You may use AI to explain an idea, help understand an error, or give a hint.

You may not submit AI-generated work that you do not understand. You are responsible for testing and explaining anything you submit.

## What if I do not understand part of a notebook?

Slow down and reread the instructions near the cell. Run the example again, look at the output, and try changing one small value to see what happens.

If you are still confused, ask for help and describe exactly which cell or idea is unclear.

## Do I have to complete every practice step?

Yes, unless your instructor says otherwise. Practice steps are included to help you build confidence before larger tasks.

Skipping steps often makes later work harder because notebooks build ideas gradually.

## What should I do before submitting work?

Rerun the notebook from top to bottom if possible. Check that required cells are complete, outputs make sense, explanations are written in your own words, and the file name or format follows the instructions.

## What if I get stuck?

Try these steps:

1. Reread the task.
2. Run the nearest working example.
3. Check one line of code at a time.
4. Make a small change and test it.
5. Write down what you tried.
6. Ask your instructor, class support, or a classmate for guidance.

Getting stuck does not mean you are bad at programming. It means you have found the next thing to practice.

## How much code should I write?

Write enough code to solve the task clearly. In this course, simple and understandable code is usually better than long or complicated code.

## What if my solution is different from someone else's?

That can be okay. Python problems often have more than one reasonable solution. What matters is that your code follows the instructions, works for the task, and is something you can explain.

## Why do I need to explain my code?

Explaining code helps show that you understand it. It also helps you notice mistakes and become more confident using Python outside the notebook.
