# Course Overview

## IT1009C: Introduction to Python for Information Technology

Welcome to IT1009C. This course introduces Python through short, guided practice activities that focus on useful, real-world information technology tasks.

No prior programming experience is required. The course is designed for beginners, including students from non-computing disciplines who want a practical first experience with Python.

## What You Will Learn

By the end of the course, you should be able to:

- write and run small Python programs
- use variables to store information
- work with numbers, text, lists, and simple data files
- use `if` statements to make decisions
- use loops to repeat useful actions
- read and reason about program output
- find, understand, and fix common errors
- explain what your code does in your own words

The goal is not to memorize every Python feature. The goal is to become comfortable using Python to solve small, practical problems.

## How the Course Is Organized

The course is built around guided notebooks. Each notebook introduces a small set of ideas, gives examples, and then asks you to practice.

Most modules include:

- short explanations
- example code to run and observe
- practice tasks
- small TODO activities
- reflection or explanation prompts
- debugging and revision opportunities

Notebook order matters because later activities build on earlier ones. Work through each notebook from top to bottom unless your instructor gives different directions.

## What You Will Do

You will practice Python by completing small tasks such as:

- calculating totals and averages
- checking whether a value meets a condition
- processing lists of names, scores, expenses, or login attempts
- reading simple CSV data
- identifying patterns in output
- fixing code that does not work yet
- improving a solution after testing it

These activities connect Python to everyday IT work, such as organizing information, checking simple rules, reviewing data, and making small decisions from clear evidence.

## What to Expect

Programming takes practice. It is normal for code to fail on the first try. In this course, errors are part of learning.

When you work on notebooks and assignments, expect to:

- run code more than once
- test small changes
- compare your output to the instructions
- revise your work after finding mistakes
- ask questions when something is unclear
- explain your thinking, not just submit code

Clear, working, understandable code matters more than complex code. A simple solution that you can test and explain is a strong solution in this course.
