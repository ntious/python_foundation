# Assignment Submission Guide

This guide explains how to prepare notebook-based assignments before submitting them. Always follow any specific directions from your instructor first.

## What to Submit

Most assignments in this course ask you to submit a completed notebook. A completed notebook usually includes:

- required code cells
- completed TODO tasks
- output from running your code
- short explanations or reflections when requested
- fix logs or notes when the assignment asks you to describe debugging work

Some assignments may also ask for a Python file, a screenshot, or another file. Use the format requested in the assignment instructions.

## Before You Submit

Open your notebook and review it from the beginning. Make sure the work is readable and complete.

Check that:

- you answered every required question
- every TODO section has been completed
- your code runs without unexpected errors
- your output matches the task or makes sense for your inputs
- your explanations are written in your own words
- any required reflection or fix log is complete

If a cell still has an error, fix it or clearly explain what you tried if your instructor allows that.

## Rerun Your Work

Before submitting, rerun the notebook if possible. This helps you catch missing variables, skipped cells, or changes that only worked because of an earlier run.

A good final check is:

1. Start at the top of the notebook.
2. Run each cell in order.
3. Stop when something does not work.
4. Fix the issue.
5. Continue until the notebook is complete.

## File Names and Format

Use the file name or format required by your instructor. If no special file name is given, keep the original notebook name or use a clear name that includes the assignment topic.

Avoid file names that are hard to understand, such as:

- `final_final_really_final.ipynb`
- `python thing.ipynb`
- `copy of copy.ipynb`

Clear file names make it easier to review your work.

## Original Work and AI-Assisted Work

Submit work that represents your own learning. Do not submit copied work from another student, a website, or an AI tool as if it were your own.

If you use AI for help, you are still responsible for the final submission. You must test the code, understand it, and be able to explain it. AI may help you learn, but it cannot replace your own practice.

## Pre-Submission Checklist

Use this checklist before turning in an assignment:

- I completed all required notebook sections.
- I ran the notebook from top to bottom.
- I checked that the output is reasonable.
- I fixed unexpected errors or explained them if allowed.
- I completed any reflection, explanation, or fix log.
- I followed the required file name and file format.
- I did not submit copied work.
- I can explain the main parts of my code.

Submitting careful, tested work is more important than submitting code that looks complicated.
