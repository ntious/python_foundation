# Fix Log (Debugging Record)

Use this log to record errors, fixes, and lessons learned while working on Python activities.

Debugging means finding and fixing problems in code. Errors are normal. The purpose of this log is to help you slow down, notice clues, and learn from each fix.

## Example Entry

### Fix Entry 1

- **Error Message:** `NameError: name 'totl' is not defined`
- **What Caused the Error:** I typed `totl` in the `print()` statement, but my variable was named `total`.
- **How I Fixed It:** I changed `totl` to `total` so the spelling matched.
- **What I Learned:** Python variable names must be spelled exactly the same way each time.

## Debugging Reminder

When something breaks:

1. Read the error message.
2. Check the line mentioned by the error.
3. Compare your code to a working example.
4. Change one thing at a time.
5. Run the code again.
6. Record what you learned.
